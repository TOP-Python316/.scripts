Алгоритм - чёткий набор действий, последовательность.
(кулинарный рецепт с чёткими шагами)

Паттерн (шаблон) - высокоуровневое описание решения.
Реализация которого будет обичаться в разных программах.
(инженерный чертёж, с решением, но не шаги по реализации).


Из чего состоят паттерны:
1. Проблема
2. Мотивация к решению проблемы, тем способом который предлагает паттерн
3. Структура классов, которые составляют решение
4. Пример на языке программирования
5. Особенности реализации в разных контекстах
6. Связи с другими паттернами


PRO:
1. проверенные решения
2. стандартизация кода
3. общий программистский словарь


CONTRA:
1. неоправданное применение паттерна
2. некоторая шаблонизация мышления


1. Порождающие паттерны нужны для гибкого создания объектов без внесения в программу лишних зависимостей.
2. Структрурные паттерны показывают способы построения связей между объектами.
3. Поведенеческие паттеры нужны для эффективной коммуникации между объектами.
